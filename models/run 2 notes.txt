What actually caused the 0.83 plateau (and what didn't)
The F1 ceiling was not an optimizer problem. Both runs plateaued at the same value despite very different optimizer configurations, which proves the bottleneck was data, not hyperparameters.
Root cause 1 — each node saw only 4 of 8 classes. When a node never sees dos, it cannot generate useful gradients for dos. FedAvg theoretically compensates by distributing the signal from the 4 nodes that do see dos, but the FedProx proximal term — especially at mu=0.10 in run 2 phase 3 — was actively restraining those 4 nodes' weights from drifting toward a good dos classifier, because that drift is exactly what the proximal term is designed to prevent. The two mechanisms were working against each other.
Root cause 2 — 3K samples per node is too thin for a 76K-parameter model. That is roughly 25 parameters per training sample per node. The model cannot fit locally, and FedAvg averages of 12 poorly fitted local models do not converge to a well-fitted global model.
Root cause 3 — XGBoost degraded the centralized baseline. RF 0.9324 > XGBoost 0.9118. RF's class_weight='balanced' handles the benign-heavy class distribution natively; XGBoost with per-sample weights and learning_rate=0.05 overfitted the majority class here.
